Web application overview

An interactive software program that runs within a web browser, much like a digital tool or service that we can access through a window rather than installing it directly onto our computer.

A web application is a single, cohesive system composed of two distinct parts working together: the Front End (the user-facing experience) and the Back End (the behind-the-scenes engine).

Front End: This is the visible interface of a website, much like the outer skin and features of a house that people see and touch from the street.

    - HTML (Hypertext Markup Language): This provides the basic structure, acting like the wooden framing and floor plan that define where rooms and walls are located.

    - CSS (Cascading Style Sheets): This handles the visual styling, serving as the paint, wallpaper, and decorations that determine how the house looks and feels.
    
    - JS (JavaScript): This creates functionality, functioning like the electrical wiring and plumbing that allow you to turn on lights, open garage doors, or use appliances.

Back End: This is the hidden server-side logic and data processing, much like the complex engine and chassis hidden under the hood of a car that makes it move.

    - Database: This is the system for storing and retrieving information, acting like a massive digital filing cabinet where every specific record is organized for quick access.

    - Infrastructure: These are the foundational hardware and network resources, similar to the network of highways, bridges, and power plants that allow the entire city to function.

    - WAF (Web Application Firewall): This monitors and filters incoming traffic, serving as the security guard at the gated entrance who checks IDs and keeps unauthorized intruders out.

How front and back end talks with each other

    - The Front End is the dining room: The customer (user) sits at the table, looks at the menu (interface), and places an order.

    - The Back End is the kitchen: The chefs (server and database) receive the order, gather ingredients, prepare the dish, and send the finished meal back out.

    - The Communication is the waiter: The waiter takes the customer's request to the kitchen and brings the requested result back to the table, ensuring the "Front End" never has to step into the messy, complicated "Back End" workspace.

    In web development, this "waiter" is usually an API (Application Programming Interface), which carries the request from your browser to the server and returns the data you asked for.

Summary:
There are many components involved in delivering a web application. Front End components like HTML, CSS, and JavaScript focus on the experience inside the browser. Back End components such as the Web Server, Database, or WAF are the engine under the surface that enable the web application to function.
